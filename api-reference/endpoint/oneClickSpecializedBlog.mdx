api: "POST https://api-v2.longshot.ai/custom/api/generate/one/click/specialized/blog"
description: "This endpoint enables you to craft a specialized compelling, factually accurate blog in just one click."
---

If no response is received here due to the extended generation time required by this API, we suggest utilizing a local API platform such as Postman or directly accessing the API through your project.

### Headers

<ParamField
  header="Authorization"
  type="string"
  placeholder="Bearer <Secret Token>"
  required
>
  Input the Authorization Bearer Token to authorize your API request.
</ParamField>

### Body

<ParamField body="topic" type="string" required>
  This input field requires the user to mention the topic they wish to write on.
  Example: "Lithium Ion discovery in India" OR "Bard: Google answer to ChatGPT"
</ParamField>

<ParamField body="description" type="string" required>
  This input field requires the user to elaborate on the topic they wish to
  write on. Example: "Showcase the future of manufacturing in India, following
  Lithium Ion discovery."
</ParamField>

<ParamField body="blog_type" type="string" default="listicle" required>
  This input field defines the type of blog you wish to generate. Following are
  the blog_types for specialized you can use:
1. **Case Study**: Analyze a scenario.
2. **Ebook**: A digital publication.
3. **Educational**: Teach a topic.
</ParamField>

<ParamField body="mode" type="string" default="serp" required>
  This field is used to decide the mode of the generation.

Modes are as follows:

1. serp
2. domain

</ParamField>

<ParamField body="domain_list" type="string">
  This field is used to input the array of the domains you want to provide as a
  reference, while generating the content. This field needs to be passed if mode
  is "domain".
</ParamField>

<ParamField body="formality" type="string" default="casual" required>
  This field is used to input formality to your generated content.

## Modes are as follows:

1. **casual**: A relaxed and informal tone.
2. **neutral**: Balanced without expressing any strong feelings or bias.
3. **formal**: Professional and follows strict conventions of grammar.

</ParamField>
<ParamField body="tone" type="string" default="witty" required>
  This field is used to input tone to your generated content.

## Modes are as follows:

1. **personable**: Helps you relate to your users on a personal level.
2. **confident**: Assures users with a tone of certainty.
3. **empathetic**: Understand and express the feelings of users.
4. **engaging**: Keeps users interested and invested.
5. **witty**: Adds humor to make users smile.
6. **direct**: Gets straight to the point for users who prefer clarity.

</ParamField>

</ResponseField>
